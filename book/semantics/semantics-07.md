# Semantics 07 — Capabilities as Actions

Capabilities define what a system can do.  
Actions define how meaning changes.

Semantic Web Design treats capabilities as **first-class semantic objects**, not as UI affordances.

## What a capability is

A capability is a declared possibility to cause change.

It specifies:

- the entity it applies to
- required inputs
- preconditions
- effects
- postconditions
- responsibility

Buttons trigger capabilities.  
They do not define them.
